# Release Notes: v1.0.0

## Summary

`v1.0.0` marks the first stable public contract release for `vuln-prioritizer`.

The tool remains a CLI for known CVEs. It does not become a scanner and it does not introduce opaque scoring. The core priority label still comes from `CVSS + EPSS + KEV`. The major release milestone is the surrounding workflow maturity:

- scanner- and SBOM-native JSON inputs
- occurrence/provenance tracking
- asset context and policy profiles
- OpenVEX and CycloneDX VEX support
- SARIF output and `--fail-on`
- static HTML reporting from saved analysis JSON
- documented JSON schemas and compatibility rules
- a composite GitHub Action for CI/CD integration

## User-visible highlights

- `analyze` now supports `auto|cve-list|trivy-json|grype-json|cyclonedx-json|spdx-json|dependency-check-json|github-alerts-json`.
- `compare` accepts the same input-format surface and remains a `CVSS-only` vs enriched comparison rather than a separate risk engine.
- `explain` surfaces applicability, provenance, asset context, ATT&CK mappings, and VEX state for one CVE.
- `data status` exposes cache timestamps and local ATT&CK metadata versions.
- `report html` renders a static executive report from a saved analysis JSON export.

## CI / OSS surface

- Root `action.yml` exposes a composite GitHub Action for `analyze` and `report html`.
- `.github/examples/` contains consumer-side examples for SARIF uploads, PR comments, and HTML artifact publication.
- `.github/workflows/release.yml` builds distributions, creates GitHub Releases, and is wired for PyPI publishing on tags.

## Contract notes

- The documented machine contract is the JSON export surface, versioned with `metadata.schema_version`.
- Bundled schemas live under [analysis-report.schema.json](https://github.com/Noetheon/vuln-prioritizer-workbench/blob/v1.0.0/docs/schemas/analysis-report.schema.json), [compare-report.schema.json](https://github.com/Noetheon/vuln-prioritizer-workbench/blob/v1.0.0/docs/schemas/compare-report.schema.json), and [explain-report.schema.json](https://github.com/Noetheon/vuln-prioritizer-workbench/blob/v1.0.0/docs/schemas/explain-report.schema.json).
- Terminal layout, Markdown layout, and warning wording remain human-facing and are not strict parsing contracts.

## Guardrails

- No heuristic or LLM-generated CVE-to-ATT&CK mappings.
- ATT&CK remains optional and file-based.
- Asset context and policy profiles affect explanation and recommendation text, not the base `priority_label`.
- VEX suppression is exact-match and occurrence-based.
