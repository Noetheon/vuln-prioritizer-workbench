# Release Notes: v1.2.0

## Focus

`v1.2.0` is the React-first Workbench release. It keeps the CLI and report contracts intact while moving the local Workbench to a packaged React/Vite UI served by FastAPI under `/app`.

## Highlights

- Added the React/Vite Workbench UI for project creation, imports, dashboard triage, findings, finding detail, governance, assets, waivers, ATT&CK coverage, run artifacts, evidence verification, provider jobs, API tokens, and vulnerability lookup.
- Kept FastAPI as the local-first backend and packaged the built React assets into the Python wheel/sdist so normal `pipx`, Docker, and installed runtime use does not require Node.js.
- Redirected `/` to `/app` while preserving legacy server-rendered Workbench paths as compatibility fallback routes during cutover.
- Added in-app API-token prompting with `sessionStorage`-only bearer storage, one-time retry for blocked mutations, and current-session cleanup when a token is revoked.
- Added stronger OpenAPI response models, standardized error envelopes for upload-limit failures, `401` bearer challenges for token auth, redacted health/provider diagnostics, `/healthz` readiness, and no-store caching for the React entry HTML.
- Hardened release gates with frontend typecheck/lint/Vitest, tracked-source checks for React/package release inputs, static asset sync, Playwright browser E2E before PyPI/TestPyPI publish, package static asset checks, isolated installed-wheel web smoke, hosted PyPI/TestPyPI web smoke, Docker SQLite/Postgres React route smoke, dependency audit, and checksum generation.

## Notes

- The static HTML report engine remains separate and continues to generate portable report artifacts.
- The Workbench remains local-first and single-workspace. Public internet deployment still requires a reviewed auth, TLS/proxy, backup/restore, retention, and operations model beyond this release.
- The CLI remains the broader automation surface for scanner/SBOM formats that are not yet exposed in the Workbench import UI.
